# 空白控制

默认情况下，Templater 的 **命令** 不会移除任何换行符，命令会被替换为对应的值，仅此而已。

但有时候我们希望在命令插入后对空白字符做一些控制，这正是这个命令工具所提供的功能。

来看个示例。下面的模板：

```
<%* if (tp.file.title == "MyFile" ) { %>
This is my file!
<%* } else { %>
This isn't my file!
<%* } %>
Some content ...
```

当条件为 false（为 true 时同理）时会输出如下内容，注意其中的空行：

```

This isn't my file!

Some content ...
```

你可能希望去掉 **执行命令** 产生的、不输出任何内容的空行。

空白控制有专门的语法：

- 标签 **起始** 处的下划线 `_`（`<%_`）会去掉命令 **之前** 的 **所有** 空白字符
- 标签 **结尾** 处的下划线 `_`（`_%>`）会去掉命令 **之后** 的 **所有** 空白字符
- 标签 **起始** 处的连字符 `-`（`<%-`）会去掉命令 **之前** 的 **一个** 换行符
- 标签 **结尾** 处的连字符 `-`（`-%>`）会去掉命令 **之后** 的 **一个** 换行符。

在我们的示例中，要修复模板去除空行，可以使用下面的模板（注意标签结尾的连字符 `-`），去掉执行命令 **之后** 的空行：

```
<%* if (tp.file.title == "MyFile" ) { -%>
This is my file!
<%* } else { -%>
This isn't my file!
<%* } -%>
Some content ...
```

会输出：

```
This isn't my file!
Some content ...
```
