# 调试设置

调试设置旨在帮助用户获取自己的设置信息并进行一些自主调试。它也可以帮助开发者确定问题的原因。

## Log Level

这是实际要记录的日志类型。
以下是不同级别及其对应的日志类型：

| Level | Types of Logs |
| ----- | ------------- |
| silent | 不记录任何错误日志，但会显示通知 |
| error | 阻止 Linter 正常运行的错误日志 |
| warn | 关于潜在问题的警告，这些问题可能无关紧要，但也可能是个问题 |
| info | 告知你发生了高层级事件的日志，例如插件已加载、已卸载、已将 moment 的 locale 设置为特定值等 |
| debug | 在调试 Linter 时有助于弄清 Linter 内部发生了什么的日志 |
| trace | 帮助确认代码中哪些区域被执行到的日志 |

!!! Note "级别建议"
    除非你正在排查 bug 或想收集要包含在 bug 报告中的信息，否则请使用 error 或 warn 作为日志级别。

## Linter Config

这是 Linter 的设置。它使得 `data.json` 中的值更易于访问。所有 bug 报告都应附上此值。

## Collecting Logs in the Linter

如果你遇到错误或某种性能问题，并希望协助确定问题所在，可以在保存时对单个文件进行 lint 或对单个文件进行 lint 时收集日志。

!!! Note "存在的日志"
    被记录的日志取决于 [Log Level](#log-level) 的值。

!!! Info "日志上报"
    存储的日志完全保存在本地。它们不会被发送到任何地方，必须由用户复制到 bug 报告中，Linter 的维护者才能看到。
