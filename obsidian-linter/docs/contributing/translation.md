# 翻译 Linter

如果你想通过翻译 Linter 来帮忙，有多种方式可以参与。
下面的说明应能帮助你完成 Linter 的翻译。

!!! note "使用翻译助手"
    在使用翻译助手进行翻译之前，你需要确保已按照[配置指南](getting-setup.md)所述将 Linter 配置为本地可用。

在对 Linter 中的译文做改动之前，我们先了解一下"翻译"是什么以及它是如何工作的。

## 什么是翻译？

Linter 的一项翻译，是一个包含 Linter 用于日志和 Obsidian 界面的文本字符串值的语言文件。语言文件位于 `src/lang/locale/` 下。每种语言用一个以其 2 字母语言缩写命名的文件表示。如果该语言是某种特定方言（例如简体中文或英国英语），请确保文件名以 2 字母语言代码开头，后跟该语言方言的 2 字母代码。

一个不带方言的常规翻译文件示例是 [en.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/locale/en.ts)。  
一个带方言的翻译文件示例是 [zh-cn.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/locale/zh-cn.ts)。

### 翻译文件的结构

翻译文件是一个 TypeScript 文件，其结构等同于一个 JSON 对象。也就是说，它是一个键值映射，其中可以嵌套值。所有翻译文件的结构都必须遵循 [en.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/locale/en.ts) 的结构——它是 Linter 中可用键的事实来源。因此，如果你需要添加一个新值或修改现有值的结构，必须先在这个文件中更新。

下面是 `en.ts` 的大致结构：

``` TypeScript
// English

export default {
  // UI text for the Obsidian commands that the Linter uses
  'commands': {...},

  // logs are general pieces of text to either partially include in the UI or to just put in the developer console.
  'logs': {...},

  'notice-text': {
    'empty-clipboard': 'There is no clipboard content.',
    'characters-added': 'characters added',
    'characters-removed': 'characters removed',
  },

  // settings.ts
  'linter-title': 'Linter',
  'empty-search-results-text': 'No settings match search',

  // lint-confirmation-modal.ts
  'warning-text': 'Warning',
  'file-backup-text': 'Make sure you have backed up your files.',

  // tabs contains tab specific UI text and setting text (tab specific setting text is text not generated by rules)
  'tabs': {
    'names': {
      // tab.ts
      'general': 'General Settings',
      'custom': 'Custom Settings',
      'yaml': 'YAML Settings',
      'heading': 'Heading Settings',
      'content': 'Content Settings',
      'footnote': 'Footnote Settings',
      'spacing': 'Spacing Settings',
      'paste': 'Paste Settings',
      'debug': 'Debug Settings',
    },
    // tab-searcher.ts
    'default-search-bar-text': 'Search all settings',
    'general': {...},
    'debug': {...},
  },

  // options is for specific UI elements that help out with settings
  'options': {
    'custom-command': {...},
    'custom-replace': {...},
  },

  // rules is where you put names and descriptions for rules and rule settings, but not dropdown record values
  'rules': {...},

  // These are the string values in the UI for enum values and thus they do not follow the key format as used above
  // they are also for dropdown records that come from union types
  'enums': {...},
};
```

可以看到，Linter 的翻译被拆分为若干分区，每个分区各有用途：

| 名称 | 说明 |
| ---- | ----------- |
| `commands` | 用于 Linter 创建的 Obsidian 命令的语言文本部分。 |
| `logs` | 用于通用文本片段的语言文本部分，这些片段可能部分出现在 UI 中，或仅写入开发者控制台。 |
| `notice-text` | 用于 Linter 向用户提示非错误类信息时所使用的通知的语言文本部分。 |
| `tabs` | 用于分页专属 UI 文本和设置文本的语言文本部分（分页专属设置文本是那些不由规则生成的文本）。 |
| `options` | 用于 Linter 自定义组件的语言文本部分，这些组件通常辅助设置。诸如自定义命令、自定义正则替换这类组件，其文本就列在此分区。 |
| `rules` | 用于规则及规则设置的名称和描述的语言文本部分，但不包含下拉项 record 的值。 |
| `enums` | 用于 UI 中枚举取值显示文本的语言文本部分，因此它不遵循上文使用的键格式。它也用于来自联合类型的下拉项 record。 |

### 部分翻译如何处理？

某种语言中缺失的任何键，都会回退到其英文取值，这使得部分翻译成为可能。这样一来，你可以随时向翻译文件中添加任意多或任意少的翻译。

## 为已有语言添加/更新值

如果你想为 Linter 在 `src/lang/locale` 下已有文件的语言添加翻译，可以手动向目标文件添加值，也可以通过运行 `npm run translate` 来完成。如果你选择使用该脚本，你可以为指定语言添加某个翻译值、列出指定语言中未翻译的键、为指定的键用新值替换已翻译的值，以及逐个翻译某种语言中所有未翻译的键。

如果你选择手动添加，则需要将 [en.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/locale/en.ts) 中的结构复制到你要为其添加值的语言文件中。

!!! note "翻译助手的前提条件"
    翻译助手要求你已按照项目要求配置好 npm，并安装好相关依赖。

译文添加完成后，就可以接受评审了。那就[提交一个 pull request](open-a-pr.md) 吧。

## 添加一种新语言的翻译

如果你想为某种尚无文件的语言添加支持，将该语言加入 Linter 需要遵循几个步骤。

### 1. 创建翻译文件

按[什么是翻译？](#what-is-a-translation)中所述确定翻译文件的名字。确定文件名后，在 `src/lang/locale/` 下为它创建一个新文件。请确保新文件遵循如下格式：

``` TypeScript
// {NAME_OF_LANGUAGE_IN_LANGUAGE_HERE}

export default {};
```

我们以添加西班牙语为例。西班牙语的 2 字母缩写是 `es`，因此新文件名为 `es.ts`。

于是我们在 `es.ts` 下创建如下文件：

``` TypeScript
// Español

export default {};
```

!!! note
    注释中的语言名称，是西班牙语中用来指代西班牙语本身的名称。
    这样做有助于母语使用者更准确地识别出该翻译所对应的语言和方言。

### 2. 更新语言助手中的导入

语言文件创建好后，它就已经存在、可供 Linter 使用了。
唯一的问题是 Linter 并不知道该文件的存在。因此我们需要更新 [helpers.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/helpers.ts)，将新文件加入它所知道的文件列表。为此，在 [helpers.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/helpers.ts) 中加入如下一行：

``` TypeScript
import {LANGUAGE_SHORT_CODE} from './locale/{LANGUAGE_SHORT_CODE}';
```

然后，需要为 [helpers.ts](https://github.com/platers/obsidian-linter/blob/master/src/lang/helpers.ts) 中的 `localeMap` 添加一条新语言的条目：

``` TypeScript
export const localeMap: { [k: string]: LanguageLocale } = {
  en,
  {LANGUAGE_SHORT_CODE},
  ...
};
```

最后，你还需要确保在 locale 到文件名的映射中加入正确的值：

``` TypeScript
export const localeToFileName: { [k: string]: string} = {
  'en': 'en',
  '{LANGUAGE_SHORT_CODE}': '{LANGUAGE_SHORT_CODE}',
  ...
}
```

!!! info "为什么要在 locale 到文件名对象中加入值?"
    这样做是为了让单元测试能够校验：Linter 所使用的每个 locale 都存在一个与之对应的唯一文件。

以添加西班牙语为例，我们会做如下改动：

``` TypeScript
import es from './locale/es'; // import new language into helper file
...
export const localeMap: { [k: string]: LanguageLocale } = {
  en,
  es, // add new language to locale mapping
  ...
};

export const localeToFileName: { [k: string]: string} = {
  'en': 'en',
  'es': 'es', // add new language locale to file name mapping
  ...
}
```

!!! note "locale map 的特殊处理"
    在某些情况下，文件所用的语言名与 Obsidian locale 所用的语言名并不相同。
    这种情况下，你需要确保将 Obsidian locale 映射到新语言：

    ``` TypeScript
    export const localeMap: { [k: string]: LanguageLocale } = {
      en,
      es,
      'pt-BR': ptBR, // special mapping of Obsidian locale to language
      ...
      'zh-TW': zhTW, // special mapping of Obsidian locale to language
      'zh': zhCN, // special mapping of Obsidian locale to language
    };

    export const localeToFileName: { [k: string]: string} = {
      'en': 'en',
      'es': 'es',
      'pt-BR': 'pt-br', // special mapping of Obsidian locale to language file name
      ...
      'zh-TW': 'zh-tw', // special mapping of Obsidian locale to language file name
      'zh': 'zh-cn', // special mapping of Obsidian locale to language file name
    };
    ```

### 3. 向文件中添加译文

翻译文件已创建、语言助手也已知悉该新语言及为其使用哪个 Obsidian locale 之后，下一步就是向这个新创建的文件中[添加译文](#addingupdating-values-for-an-existing-language)。

### 4. 提交 pull request

译文添加完成后，新语言的引入就应当可以接受评审了。那就[提交一个 pull request](open-a-pr.md) 吧。
